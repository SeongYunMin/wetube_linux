<위튜브 클론 코딩 필기록>
node.js 파일을 만들기 위해서는 파일명을 package.json으로 해줘야 한다
이렇게 하면 nodeJS 프로젝트 만들 수 있게 된것이다 json 형태로 구성된 파일이다
일단 {}로 json 오브젝트를 만든후 name을 써서 파일의 이름을 지어주자
json은 프로그래머가 파일에 정보를 저장하기 위해 만든 방식 중 하나이다
nodeJS의 경우 파일명은 무조건 package.json이어야 한다 무조건 소문자로!!
그리고 깃 repository를 만들 건데 깃허브 데스크탑을 쓰는 것이 아니라
우분투를 쓸 것이다 우선 wetube 디렉토리에 git init을 입력해서 wetube폴더를
깃 폴더로 만들고 깃허브 페이지에서 wetube 레포지토리 생성한후 그 url을 복사하도록 하자
그리고 그 것을 wetube 디렉토리에서 git remote add origin url 입력하여 깃허브와
연동이 되도록 하자
사실 package.json은 우리가 직접 타이핑해서 초안을 만드는것이 아니라 npm을 써서
쉽게 만들 수 있다 콘솔창에 npm init 치고 엔터를 계속 치면 package.json의 초기설정을
에러없이 쉽게 만들 수 있다 이렇게 한 후 package.json 들어가보면 오브젝트 안에 여러 내용이
있음을 알 수 있다  노드js는 오직 2개의 파일로 시작한다
바로 package.json 과 index.js 이다

<윈도우 개발 환경 강의 필기록>
{3_3 prettier and more command}
아 한가지 명심해야 되는 거는 지금 우리가 코드를 짜고 있는 이 VSC 에디터는 기존에 윈도우 환경
에서 쓰던 에디터와 아얘 다른 거라고 생각해야 한다 얘는 WSL, 즉 우분투 환경에서 쓰는 에디터이다
따라서 prettier도 없고 기존에 설치해뒀던 윈도우 한정 익스텐션은 다 없는 것을 알 수 있다
리눅스 환경에서 prettier를 조작하는 것은 노마드코더 원도우 개발환경 강의를 참고하자
두 환경에서 모두 쓸 수 있는 에디터가 VSC이다 설정 들어가보면 윈도우랑 우분투 설정이 나눠져 있음을
알 수 있다 설정할때 어떤 환경에서 코딩하는지 인지하고 설정을 해주면 된다
리눅스에서 쓰는 몇몇 추가적인 커맨드에 대해 알아보자면
mkdir: 디렉토리 생성 
mv A B: A를 B로 이동 이때 A,B가 같은 디렉토리이면 파일의 이름을 A에서 B로 바꾼다
rm: 파일을 지운다(디렉토리, 즉 폴더는 지울 수 없다)
rm -rf: remove folder의 약자로 얘는 폴더를 지울 수 있게 된다
